Somma Cipher
SPEC v0.2 — PANORAMICA

Somma Cipher è un algoritmo di cifratura testuale basato su quattro principi:

  • Codifica modulare — ogni carattere diventa N tale che N mod m = indice
  • Numeri pseudo-casuali — lo stesso carattere produce numeri diversi ogni volta
  • Junk letters — numeri extra aumentano l'entropia e mascherano la struttura
  • Posizione rumore = password — solo chi conosce la chiave sa cosa ignorare

Versione 0.2: encoding estesa a 617 caratteri (alfabeto greco, cirillico, emoji, simboli). Il flusso dati è continuo, senza separatori — ogni token è esattamente 4 cifre decimali.

Magic String
SEZIONE 01

Ogni file inizia con:

5EBAF12E

Identifica il file come Somma Cipher e previene decifrature su dati non validi.

Encoding
SEZIONE 02

Lista ordinata di 617 caratteri. L'indice è il valore numerico del carattere. Include ASCII, accenti latini, greco completo, cirillico completo, simboli matematici, emoji.

N mod 617 = indice del carattere

N è sempre un numero a 4 cifre (0000–9999). Non viene usato alcun separatore.

Cifratura
SEZIONE 03
1) n = index(c) in ENCODING
2) N casuale tale che N mod m = n, con 0 ≤ N ≤ 9999
3) N scritto come 4 cifre decimali (0000–9999)

Il flusso è la concatenazione di tutti i numeri a 4 cifre, senza alcun delimitatore:

0004019841230004889201280634...
Password
SEZIONE 04

La password agisce su due livelli indipendenti:

1 — Shift dell'indice (keystream Vigenère)

Per ogni carattere reale alla posizione ci:

k0 = index(P[ ci      mod Lp ])
k1 = index(P[(ci+1)  mod Lp ])
k2 = index(P[(ci+2)  mod Lp ])
shift = (k0×3 + k1×7 + k2 + (k0 XOR k1 XOR k2)) mod m

cifra:   N codifica  (n + shift) mod m
decifra: n = (N mod m − shift + m) mod m

Senza la password corretta N mod m restituisce un indice sbagliato → carattere sbagliato → testo incomprensibile.

2 — Posizione delle junk letters

Controlla il junk_interval come descritto nella sezione 06. Tutti i caratteri della password devono essere presenti nell'ENCODING.

Junk Letters
SEZIONE 05

Numeri casuali 0000–9999 senza vincolo modulare. Indistinguibili dai dati reali senza la password.

Frequenza Junk
SEZIONE 06
base_interval = max(4, floor(sqrt(L)))
offset = (index(P[i mod Lp]) + index(P[(i+1) mod Lp])) mod 7
junk_interval = base_interval + offset

Ricalcolato ad ogni junk inserita.

Struttura File
SEZIONE 07
5EBAF12E
{L}
{STREAM 4 cifre senza separatori}

Esempio:

5EBAF12E
42
0312051900280634...
Decifratura
SEZIONE 08
1) Verifica "5EBAF12E"
2) Leggi L
3) Ricostruisci sequenza junk_interval dalla password
4) Leggi stream a blocchi di 4 cifre
5) Salta le posizioni junk
6) n = N mod m → c = ENCODING[n]
Proprietà
SEZIONE 09
  • Ridondanza numerica — stesso carattere, numeri diversi
  • Distribuzione modulare — frequenza mascherata
  • Rumore password-dipendente
  • Flusso continuo senza blocchi espliciti
  • Nessun separatore — nessun carattere è "speciale"
